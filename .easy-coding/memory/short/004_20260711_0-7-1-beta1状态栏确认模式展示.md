## 技术记忆候选

- 架构 / 接口决策：用户可见状态栏只消费 `snapshot_state()` 已解析的 `effective_confirm_mode`，不重复实现项目模式与 session 覆盖优先级。

- 实现模式 / 复用写法：以动态品牌前缀 `Easy Coding [Mode]` 拼接 Ready、Waiting 和任务状态后缀，可确保所有状态分支格式一致。
- 易错点 / 修复策略：SemVer 字符串 `0.7.1-beta.1` 低于历史 `0.7.1-beta0`，会让 upgrade 把旧项目误判为更新版本；同核心延续历史命名时应使用 `0.7.1-beta1`，并用 `compareVersions("0.7.1-beta0", "0.7.1-beta1") === -1` 固化回归。
- 验证经验：版本改动必须同时验证 package、lockfile、介绍页、CHANGELOG、全量测试、构建和 npm pack dry-run；仅看打包成功无法证明升级顺序正确。
