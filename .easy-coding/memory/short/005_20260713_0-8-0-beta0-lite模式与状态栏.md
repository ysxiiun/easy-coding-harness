## 技术记忆候选

- 架构 / 接口决策：`lite` 与 `guard` 共用 `ANALYSIS → IMPLEMENT`、`VERIFICATION → MEMORY` 两个确认门，但状态 API 在 Lite 下拒绝 `IMPLEMENT → REVIEW`，只允许代码主链进入 VERIFICATION。
- 工程规则 / 工作流：若切换到 Lite 时遗留 `IMPLEMENT → REVIEW` pending edge，hook 注入 `lite-review-bypass-required` breadcrumb，ec-workflow 必须先取消旧边再自动进入 VERIFICATION。
- 实现模式 / 复用写法：状态栏只消费 `snapshot_state().effective_confirm_mode`；品牌渲染为 `**Easy Coding**`，模式独立渲染为 `**Mode**`，任务、阶段和 Handoff agent 继续使用行内代码背景。
- 易错点 / 修复策略：完整状态栏示例内部已有反引号时，不能再用单反引号包住整条字符串；CommonMark 的代码 span 不支持这种嵌套，方案文档应使用 fenced code block 或直接输出 blockquote。
- 验证经验：确认模式升级必须同步状态 API、skills、主约束、CLI 配置入口、三平台安装测试、状态快照、文档、版本文件和 npm pack 内容检查。
