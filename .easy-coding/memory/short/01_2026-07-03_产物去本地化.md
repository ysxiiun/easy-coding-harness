## 技术记忆候选

- 架构原则：harness 所有 git 共享产物必须"去本地化"——无本机绝对路径、无运行时编译产物。审计要点覆盖 hook 命令、install-manifest、config.yaml、主约束文件、supermodule 父子拓扑、task 文件。
- 易错点·绝对路径泄漏：`createProjectInitTask` 曾把 `params.cwd`(绝对仓库根)写进 `.easy-coding/tasks/project-init/task.json` 的 `context.project_path`;该字段**零消费方**(全仓 `rg project_path` 只有写入点),而 `tasks/` 会被提交(gitignore 只忽略 `sessions/`),团队 pull 即泄漏。根治=直接删字段+参数。
- 实现模式·存量清洗：`stripInitTaskProjectPath(cwd)` 幂等剥离旧 `project_path`,挂在 upgrade 每个 target(与 `setPendingInitSince` 同处)。
- 易错点·编译产物：hooks 是多文件互相 import 的 Python 包(`easy_coding_state.py` 为核心库,4 个入口都 import 它);launcher 用 `runpy.run_path` 执行入口,被 import 的库在 hooks 目录旁落 `__pycache__/*.pyc`。
- 实现模式·编译产物根治(最优)：launcher 首行设 `sys.dont_write_bytecode=True`,从源头不生成;改 launcher 会变 base64,upgrade 据"项目相对 hook 路径"识别托管注册并自动刷新存量。
- 实现模式·兜底：`ensureHookBytecodeIgnored` 向项目 `.gitignore` 追加 `__pycache__/`,挂在 init/add-agent/upgrade 三入口(紧邻 `ensureEasyCodingSessionsIgnored`);Python3 字节码统一进 `__pycache__/`,一条足够。
- 易错点·biome noDelete：不能用 `delete obj.key`;改 `obj.key = undefined`,`JSON.stringify` 会丢弃 undefined 值的键,持久化后同样不含该字段,幂等不受影响。
- 验证经验·flaky：跑真实 python 的重集成测试(claude/codex-qoder/upgrade 里的 hook 执行用例,单个 7~23s)全量并发时会因 CPU 抢占超时假阳性;判定回归前先**单独复跑**确认。勿把 lint 与 test 放同一批并行发,会加剧抢占。
- 参考·harness 自身已干净：root `.gitignore` 已忽略 `__pycache__/`+`*.pyc`,`copy-templates.mjs:11` 过滤 `__pycache__`,故 npm 包不含 pyc,问题只在用户侧。
